博客，不仅是一个记录我想法的地方，而且也是一个我和外界沟通的渠道，期望有人能在我的博文下与我互动，碰撞出新的火花，所以博文可以被评论势在必行。在寻求其他方式的时候，Utterances 不经意间进入了我的视野，它轻量，配置简单，且也可以利用 GitHub 来搭建，于是就它了。

安装 Utterances App

步骤一

首先，请确保已注册 Github。注册完毕之后或你已有账号请前往步骤二。

步骤二

创建所需的仓库。

由于我已经创建了 hugo-comment ，这里用 hugo-comments 做演示。你创建的时候取你喜欢的名字就好。

步骤三

在 GitHub 上安装 Utterances App。

进入 Utterances App 主页，点击安装。

因为我已经安装过，所以这里是配置按钮。

访问仓库选择刚刚创建的 hugo-comment ，点击安装（因为我已经安装过，所以这里是保存按钮）。

配置

步骤一

在 config.toml 配置中新增如下配置。

disable_comments = false
[params.utteranc]
    repo = "{Github Name}/{Repo Name}"
    issueTerm = "title" # 设置每篇文章对应的 issue 的名字
    theme = "github-dark"

其中 {Github Name} 为 GitHub 账户名（username），{Repo Name} 为刚刚创建的仓库名，如果没有配置对，评论框是显示不出来的。

步骤二

创建 HTML 模板。

在 theme/your theme name/layouts/partials 文件夹下创建评论模板。

新建名如 utteranc.html 的文件，文件内容为：

{{- if isset .Site.Params "utteranc" -}}
  {{- if and (isset .Site.Params.utteranc "repo") (not (eq .Site.Params.utteranc.repo "" )) (eq (.Params.disable_comments | default false) false) -}}
  <script src="https://utteranc.es/client.js"
    repo= "{{ .Site.Params.utteranc.repo }}"
    issue-term="{{ default "title" .Site.Params.utteranc.issueTerm }}"
    theme="{{ default "github-light" .Site.Params.utteranc.theme }}"
    crossorigin="anonymous"
    async>
  </script>
  {{- end -}}
{{- end -}}

步骤三

配置显示位置。

打开页面模板，以我的主题为例为 single.html 文件。在 <footer></footer> 标签内添加 {{ partial "posts/utteranc.html" . }} 标签内容。

以上，如果不出意外，评论系统就配置完成。

hugo server -D 启动浏览看看吧。

个性化

主题

支持以下六种类型主题，一种明亮主题，物种暗黑主题。可以按照自己喜好在 config.toml 中配置。

  • github-light
  • github-dark
  • github-dark-orange
  • icy-dark
  • dark-blue
  • photon-dark

issue-term 映射配置

支持以下几种：

  • pathname
  • url
  • title
  • og:title
  • issue-number
  • specific-term

个人推荐使用 title ，文章标题一般是不变的，可以很方便地查看每篇文章对应的评论。
