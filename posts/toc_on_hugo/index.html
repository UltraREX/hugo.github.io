总所周知，一本书拥有目录是一件非常重要的事情。不仅可以让人非常快速的了解到这本书所讲的内容，而且可以非常高效的查阅。而一篇文章也是如此。 查阅 Hugo 官方文档 后，发现它在支持这一特性，并且有两个例子可供我们参考。

配置

步骤一

在 config.toml 配置中新增如下配置。

[params]
    # Table of Contents
    toc = true
    wordCount = 0

其中 wordCount 为文章字数大于该值时才生成目录配置。

步骤二

创建 HTML 模板。

在 layouts/partials 文件夹下创建文章目录模板。

新建名如 toc.html 的文件，文件内容为：

{{ if and (gt .WordCount .Site.Params.wordCount ) (eq (.Site.Params.toc | default false) true) }}
<aside class="post-toc-absolute" id="post-toc">
    {{ .TableOfContents }}
</aside>
{{ end }}

步骤三

配置显示位置。

打开页面模板，以我的主题为例为 single.html 文件。在合适的位置插入 {{ partial "toc.html" . }} 。以我的博客为例，我放在了文章正文所在 div 前，方便以后控制样式和位置。

步骤四

配置样式。

在 static 文件夹下创建 css 文件夹，在 css 文件夹内新建 toc.css 文件。文件内容为：

.post-toc-absolute {
    position: absolute;
    right: 4%;
}

.post-toc-fixed {
    position: fixed;
    top: 20px;
    right: 4%;
}

/* Media Queries */
/* 简单用设备屏幕尺寸来判断是否是移动端 */
@media (max-width: 1080px) {
    .post-toc-absolute {
        display: none;
    }
    .post-toc-fixed {
        display: none;
    }
}

并在 config.toml 中配置使用：

[params]
    # Custom CSS
    custom_css = ["css/toc.css"]

步骤五

设置滚动跟随。

在 static 文件夹下创建 js 文件夹，在 js 文件夹内新建 toc.js 文件。文件内容为：

let toc = this.document.getElementById("post-toc");
let fixedClsName = "post-toc-fixed";
let absoluteClsName = "post-toc-absolute";
window.addEventListener(
    'scroll',
    function (e) {
        let scroll_height = window.scrollY;
        if (scroll_height > 315) {
            if (toc.className != fixedClsName)
            {
                toc.className = fixedClsName;
            }
        } else {
            if (toc.className != absoluteClsName)
            {
                toc.className = absoluteClsName;
            }
        }
    }
);

这样，在文章打开时，目录在页面的某个位置。滚动页面，目录随页面一起滚动，当目录头部靠近浏览器窗口顶部时，则固定显示在顶部。你可以在本页面滚动感受一下。
